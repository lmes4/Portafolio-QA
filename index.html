Proyectos

Pagina Web – QA Manual y Automation

Realicé el aseguramiento de la calidad de una aplicación web ya desarrollada, trabajando en equipo bajo metodologías ágiles Scrum. La gestión del trabajo se realizó mediante sprints semanales utilizando Jira, participando activamente en las ceremonias de Sprint Planning, dailys y Sprint Review al final de cada sprint.

Diseñé las pruebas de las historias de usuario siguiendo la metodología BDD, partiendo de los criterios de aceptación definidos. Se crearon escenarios en lenguaje Gherkin para disponer de pruebas legibles y comprensibles para todo el equipo, gestionándolas con AssertThat para mantener la trazabilidad entre requisitos, pruebas y resultados dentro de Jira. Las pruebas se ejecutaron por historia de usuario y se registraron los resultados en entornos web y móviles.

Durante cada sprint se realizaron pruebas funcionales y, al final del mismo, pruebas exploratorias para identificar defectos no cubiertos por los casos definidos. Los errores detectados se reportaron en Jira, vinculándolos a la historia de usuario correspondiente para asegurar la correcta trazabilidad entre requisitos, pruebas y defectos.

Al cierre de cada sprint se preparó la release en Jira y se documentó en Confluence la información relacionada con la entrega, incluyendo la documentación completa del plan de pruebas, resultados de ejecución y estado de calidad del producto.

Tras la release, se identificaron los flujos principales de la aplicación y se automatizaron las pruebas de regresión utilizando Playwright y Pytest con Python. Estas pruebas se integraron en un flujo de integración continua con GitHub Actions, configurado para ejecutar automáticamente la regresión de forma semanal, alineada con el cierre de cada sprint.

Portafolio QA – Desarrolladora Web y QA en Testing de APIs

Desarrollé un portafolio web utilizando HTML5, CSS3 y JavaScript, integrando un formulario conectado a una API generada con Supabase. Se diseñaron las tablas de la base de datos y se realizaron consultas en SQL para la inserción y consulta de datos de prueba del formulario.

Realicé testing de APIs utilizando Postman, creando pruebas para las peticiones GET (consulta), POST (creación), PATCH (edición) y DELETE (eliminación). Se implementaron tests que validan códigos de respuesta positivos, negativos y de autenticación, así como la correcta estructura y contenido de las respuestas.

Además, se implementó un flujo de integración continua con GitHub Actions, configurando Newman para ejecutar automáticamente las pruebas creadas en Postman en cada cambio del repositorio.
