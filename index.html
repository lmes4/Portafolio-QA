Proyects

Pagina Web – QA Manual y Automation

I performed quality assurance on an already developed web application, working as part of a team under agile Scrum methodologies. Work management was carried out through weekly sprints using Jira, actively participating in Sprint Planning, dailys, and Sprint Review ceremonies at the end of each sprint.

I designed tests for user stories following the BDD methodology, based on the defined acceptance criteria. Scenarios were created in Gherkin language to ensure tests were readable and understandable for the entire team, managing them with AssertThat to maintain traceability between requirements, tests, and results within Jira. Tests were executed per user story, and results were recorded in both web and mobile environments.

Durante cada sprint se realizaron pruebas funcionales y, al final del mismo, pruebas exploratorias para identificar defectos no cubiertos por los casos definidos. Los errores detectados se reportaron en Jira, vinculándolos a la historia de usuario correspondiente para asegurar la correcta trazabilidad entre requisitos, pruebas y defectos.

During each sprint, functional testing was carried out and, at the end of each sprint, exploratory testing was performed to identify defects not covered by the defined test cases. Detected issues were reported in Jira, linking them to the corresponding user story to ensure proper traceability between requirements, tests, and defects.

After the release, the application’s main flows were identified and regression tests were automated using Playwright and Pytest with Python. These tests were integrated into a continuous integration workflow with GitHub Actions, configured to automatically run regression tests on a weekly basis, aligned with the end of each sprint.

Portafolio QA – Web Developer and QA Engineer in API Testing

I developed a web portfolio using HTML5, CSS3 and JavaScript, integrating a form connected to an API generated with Supabase. Database tables were designed, and SQL queries were performed for inserting and retrieving test data from the form.

I performed API testing using Postman, creating tests for GET (retrieval), POST (creation), PATCH (update), and DELETE (deletion) requests. Tests were implemented to validate positive, negative, and authentication response codes, as well as the correct structure and content of the responses.

In addition, a continuous integration workflow was implemented using GitHub Actions, configuring Newman to automatically run the tests created in Postman on every repository update.
